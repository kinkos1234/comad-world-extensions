# Taste Rubric — 자기비평 점수 기준 (6축)

> CRITIQUE 단계에서 렌더 스크린샷을 6축으로 채점(각 0~5). 합계 24/30 미만이면 REGENERATE.
> 점수는 "느낌"이 아니라 아래 구체 기준으로. 낮은 축을 다음 라운드의 수정 타깃으로.

| 축 | 0~2 (slop) | 3 (보통) | 4~5 (탑티어) |
|---|---|---|---|
| **1. 색 절제 (Restraint)** | 무지개/보라그라데·다색 | 베이스+2색 | 무채 베이스 + 단일 절제 액센트(면적<5%) |
| **2. 타이포 (Type)** | 균일 크기·800두께·tracking 0 | 스케일 있음 | 큰 대비·600디스플레이·-0.03em·정렬된 baseline |
| **3. 공간·정렬 (Space)** | 전부 중앙·균일 여백 | 일부 정렬 | 8px 리듬·비대칭·optical 정렬·후한 hero 여백 |
| **4. 표면·디테일 (Surface)** | 회색보더·그림자남발·큰radius | 무난 | hairline·미세그림자·일관 radius위계·inner highlight |
| **5. 신뢰감 (Realism)** | 플레이스홀더·빈박스·이모지 | 일부 실콘텐츠 | 실제 제품 목업·실데이터 숫자·고유명사·0 이모지 |
| **6. 차별성 (Distinct)** | 템플릿 그 자체 | 약간 특색 | "이 브랜드만의 것"·레퍼런스 옆에 둬도 안 밀림 |

## 적용
1. v1 렌더 → 6축 채점 + 각 축 한 줄 근거.
2. 합계 ≥24 AND 모든 축 ≥3 AND anti-slop 즉시탈락 0개 → **통과**.
3. 미달 → 가장 낮은 2축을 구체 수정 지시로 → REGENERATE → 재채점.
4. 최대 3라운드. 3라운드 후에도 미달이면 사용자에게 "여기까지 + 막힌 축" 보고(무한루프 금지).

## 점수 기록 포맷 (생성 로그에 남김)
```
v1: 색4 타입3 공간3 표면3 신뢰2 차별3 = 18/30 → 신뢰·공간 타깃
v2: 색4 타입4 공간4 표면4 신뢰4 차별4 = 24/30 → 통과
```

> 핵심: 이 루브릭은 **개별 미감 점수**다. comad-qa-evidence(기능 QA)·design-review(시각 일관성)와 별개·보완. Taste Layer 는 "좋아 보이나"를, qa-evidence 는 "동작하나"를 본다.
